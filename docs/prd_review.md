## 🔴 致命问题：模型规格数据根本性错误

**Agent 5（模型接口师）发现，PRD 的核心约束假设全部错误：**

| 参数 | PRD 声称 | 实际规格 | 偏差 |
|:---|:---|:---|:---|
| 上下文窗口 | 20,000 tokens | **262,144 tokens** | 低估 **13倍** |
| 最大输出 | 8,192 tokens | **65,536 / 32,768 tokens** | 低估 **4-8倍** |
| 日成本 | ¥4.42 | **¥0.38-0.61** | 高估 **7-12倍** |
| 月成本 | ¥97 | **¥8-15** | 高估 **6-12倍** |

### 影响：整个架构需要重新设计

PRD 声称"20,000 tokens 上下文限制是系统设计的核心约束"，并据此设计了三级渐进压缩。但实际 262K 窗口下：

- 全天 6-10 万汉字 ≈ 33,000-67,000 tokens，**仅占实际窗口的 13%-26%**
- 90% 的工作日可以**单次调用处理全量文本**，无需分片
- 三级压缩架构可简化为**两级弹性架构**，API 调用从 21+ 次降至 1-4 次

### 建议的新架构

| 场景 | 条件 | 策略 | 调用次数 |
|:---|:---|:---|:---|
| 短日（90%场景） | 文本量 < 80K tokens | 全量单次输入 → 直接生成日报 | 1 次 |
| 长日（10%场景） | 文本量 80K-250K tokens | 按时段分 2-3 片 → 合并生成日报 | 3-4 次 |
| 降级路径 | 模型不可用/降级小模型 | 保留原三级架构 | 21+ 次 |

## 🟡 六团队共识问题（多个 Agent 独立指出）

### 1. 对话形态假设偏差（Agent 1 + Agent 2）

PRD 隐含假设对话是"长时间连续会议"，但实际工作日：
- **60%+ 对话不到 10 分钟**（走廊聊天、电话、快速确认）
- 对话时长呈强双峰分布，而非连续分布

**共识建议**：增加"对话会话检测"作为分片前置步骤（按静默 >3min + 说话人变化切分），短对话聚合打包处理。

### 2. 评分体系过于复杂且不完备（Agent 3 + Agent 4）

| 问题 | 详情 |
|:---|:---|
| 两套评分公式关系不明 | 四维度公式（§1.1.3）与 RuleScore/ModelScore 二元公式（§4.3.1）如何嵌套？ |
| β/γ/δ 权重未定义 | 四维度公式中只定义了 α，其余三个权重完全缺失 |
| 15+ 参数无冷启动方案 | 如何确定初始值？无历史数据时怎么办？ |
| DurationScore >30min 衰减不合理 | 长会议不一定不重要，指数衰减过于激进 |

**共识建议**：先做 MVP 简化版（`Score = KeyPersonBase + LLMScore × 10`，仅 2 个变量），积累数据后迭代。

### 3. Sentence-BERT 中文 ASR 口语效果存疑（Agent 2 + Agent 4）

- ASR 口语文本噪声大（缺标点、识别错误、填充词密集），Sentence-BERT 嵌入质量退化
- 中文摘要句式高度模板化，0.85 相似度阈值易误合并不同信息
- 具体模型未指定（Sentence-BERT 是架构族，非具体模型）

**共识建议**：
- 去重阈值提高至 0.90-0.92
- 优先用"说话人切换 + 静默间隔"作为分片边界信号，语义检测作为辅助
- 明确选用 text2vec-base-chinese 或 m3e-large

### 4. 置信度阈值 0.6 对关键人不安全（Agent 1 + Agent 6）

P0 关键人可能因低语、快速交代而置信度偏低，统一 0.6 阈值会丢弃关键指示。

**共识建议**：分级阈值——P0/P1 降至 0.35-0.4，非关键人保持 0.6。

### 5. 测试数据缺少 ASR 噪声模拟（Agent 6 + Agent 1）

PRD 只定义了场景和时长分布，完全没有 ASR 错误模式模拟（同音字、断句错误、漏词、说话人标签错误）。

**共识建议**：构建参数化 ASR 噪声注入器，测试数据必须包含真实错误分布。

### 6. 文档内部自相矛盾（Agent 4 发现）

| 位置 | 矛盾 |
|:---|:---|
| 压缩比 | 正文说"100:1以上"，三级乘积=150:1，评估指标写 30-50:1 |
| 输出格式 | §1.1.5 列 4 个模块，§5.3.3 列 6 个模块，字段不一致 |

## 各 Agent 独立发现的重要问题

### Agent 1（数据工程师）

| 问题 | 建议 |
|:---|:---|
| 200ms 对齐粒度过细 | 放宽至 500ms，匹配 Speaker Diarization 实际分辨率 |
| 合并后中位数 45-90s 偏长 | 实际经验 20-60s，增设 120s 合并上限 |
| 设备输出格式过于理想化 | 先获取真实 API 响应样本验证，增加原始数据快照层 |
| 有效文本量高估 | 考虑 40-60% 有效说话比例，实际 3-8 万汉字 |
| 数据模型缺少不确定性字段 | 增加 speaker_confidence、asr_confidence、alignment_quality |

### Agent 2（策略引擎师）

| 问题 | 建议 |
|:---|:---|
| 关键人保护区"取代"机制过于激进 | 改为"优先约束"，允许内部标记低价值段 |
| 10-20% 重叠区域 ROI 低 | 取消原文重叠，改为片段摘要携带"尾部上下文"字段 |
| 保护区 >60min 内部细分规则不完备 | 补充具体算法、数据结构、累积损失评估 |
| 多保护区并行/嵌套规则缺失 | 定义完整仲裁决策树 |
| 碎片短对话处理缺失 | 增加碎片聚合策略，<5min 非关键人打包处理 |

### Agent 3（评分算法师）

| 问题 | 建议 |
|:---|:---|
| P0 保障分 80 可能饱和 | 引入二次排序或每日配额，避免闲聊片段挤占 |
| 连续性保护加成幅度未定义 | 定义公式：bonus = neighbor_score × 0.3 × max(0, 1-interval/5) |
| 时段系数硬编码不灵活 | 改为可配置参数，支持跨时区/非标准作息 |
| Top-K / 阈值参数缺失 | 给出默认值（K=20 或阈值=60），与 token 预算联动 |
| 评分可解释性不足 | 每片段保留评分分解记录 |

### Agent 4（摘要生成师）

| 问题 | 建议 |
|:---|:---|
| 三级压缩误差累积不可逆 | 引入"关键事实直通通道"——决策/行动项/关键数据直通全天级 |
| 跨时段同一议题无法关联 | 引入 topic_id 贯穿机制 + "悬置议题"字段 |
| tomorrow_preview 幻觉风险高 | 降级为"明日已知安排"，仅基于明确提及的时间承诺 |
| 反幻觉策略与 ASR 误差矛盾 | 引用标记应包含 ASR 置信度，低置信度标注"待确认"而非引用错误文本 |
| 日报 2000-3000 字偏长 | 提供分层阅读体验：30秒版(100字) / 3分钟版 / 完整存档版 |
| 层级间溯源链不完整 | 每级输出都应包含 source_refs 字段 |

### Agent 5（模型接口师）

| 问题 | 建议 |
|:---|:---|
| 模型快照版本未锁定 | 使用 qwen3-max-2026-01-23 而非 qwen3-max |
| 未利用 Batch API | 离线场景用 Batch 可节省 50% 成本 |
| 思考模式成本不可控 | 设置 thinking_budget（时段级 8192，全天级 16384） |
| 流式处理在离线场景价值有限 | 默认用 Batch，流式作为可选能力 |

### Agent 6（配置测试师）

| 问题 | 建议 |
|:---|:---|
| L3 内容推断 70-80% 准确率 | 标记为"疑似"，赋予折扣权重（P0 疑似 = P1 待遇） |
| Speaker ID 冷启动路径缺失 | 定义冷启动阶段（前 5 天），降低过滤阈值，积极提示用户确认 |
| 热更新 5-10s 过度设计 | 简化为"启动时加载 + 手动 reload" |
| 版本控制自建无必要 | 直接用 Git 管理 key_people.yaml |
| ROUGE-L/BERTScore 不够 | 增加关键信息召回率、行动项 F1 作为一级指标 |
| 边界测试遗漏场景 | 补充：语言混合、跨天录音、多设备同一会议、ASR 质量极差 |

1. **先修正 PRD 中的模型规格数据**，这是一切的基础
2. **基于正确的 262K 窗口重新设计架构**，确定两级弹性方案的具体参数
3. **简化评分体系为 MVP 版本**，2 个核心变量先跑通闭环
4. **获取真实设备 API 响应样本**，验证数据格式假设
5. 完成以上修订后，Phase 1 的三个 Agent（数据工程师 + 模型接口师 + 配置测试师）即可开工
